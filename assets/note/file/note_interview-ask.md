# 🎯 前端工程師面試必問問題清單（Vue3 / TypeScript）

面試除了回答問題，更重要的是了解公司文化、技術成熟度與實際工作情況。以下是專為本職缺整理的必問項目，協助你判斷是否適合。

---

## 🧱 1. 專案技術架構

**🔧 請說明目前的前端技術棧：**
- Vue3  
- TypeScript  
- PrimeVue  
- 其他主要第三方套件？

**📌 其他細節：**
- 套件更新策略是什麼？多久更新一次？
- 是否有前端架構規範（composables、store、component 組織）？
- API 使用的後端技術是什麼？（如 .NET）
- 前後端協作方式（Swagger、文件、定期對齊等）

---

## 👥 2. 團隊與分工

**🧑‍💻 前端團隊規模：**
- 前端目前有多少人？
- 每人負責的模組或領域？

**🎨 UI/UX 與 PM：**
- 是否有 UI/UX 設計師？
- 設計稿完整度如何？  
- PM 是否提供明確規格？

---

## 📚 3. 工作內容與專案型態

**📌 工作內容比例：**
- [ ] 新功能開發：___%  
- [ ] 舊系統維護：___%  
- [ ] 切版工作：___%  
- [ ] 客製化表單或後台：___%

**其他：**
- 主要專案類型是 B2B、後台系統、官網還是 SAAS？
- 是否同時負責多個專案？

---

## ✈️ 4. 出差相關（非常重要）

**請具體說明：**
- 出差目的？（例如：客戶端導入、教育訓練、裝機、部署）
- 頻率與每次時間長度？
- 是否提供出差補助？包含：
  - [ ] 交通  
  - [ ] 住宿  
  - [ ] 餐費  
- 出差時間是否會延長工時？是否能補休？

---

## ⏰ 5. 加班文化與制度

- 平常是否會固定加班？  
- 專案趕工時的頻率？  
- 加班是否提供：
  - [ ] 加班費  
  - [ ] 補休  
- 專案時程如何規劃？會不會經常臨時更改需求？

---

## 🚀 6. 開發流程與 CI/CD

**開發流程是否具備：**
- [ ] PR Review  
- [ ] 程式碼規範（ESLint / Prettier）  
- [ ] CI/CD（GitHub Actions / Azure DevOps / GitLab CI）  
- [ ] 多環境流程（Dev → Staging → UAT → Prod）  
- [ ] 自動化測試（如可略，但問了會加分）

**需求流程：**
- PM 是否提供明確的使用者故事與規格文件？

---

## 📦 7. 前端日常協作方式

- 是否有 Code Style 或架構指南？
- 版本控制流程（Git Flow / trunk-based）？
- 一個任務從需求 → 設計 → 開發 → 測試 → 上線的完整流程？
- 是否可參與技術決策或改善流程？

---

## 📈 8. 學習與升遷

- 是否提供：
  - [ ] 在職教育訓練  
  - [ ] 技術研討會補助  
  - [ ] 線上課程補助  
- 前端的職涯發展路線為何？
- 評估績效的方式是什麼？

---

## 🖥️ 9. 工作設備

- 公司提供的電腦規格？可否選擇 Windows / Mac？
- 是否提供雙螢幕？
- 是否提供工具授權，如：
  - [ ] JetBrains  
  - [ ] Figma  
  - [ ] GitHub Private Repo  

---

## 📄 10. 其他需要確認的事項

- 試用期長度？與正式待遇是否相同？
- 是否會調整工作內容或地點？
- 出差與加班是否計入績效？

---

# ✅ 建議用法

📌 把這份清單放進手機或 Notion  
📌 面試尾聲時挑最重要的 5–7 題問  
📌 盡量使用「請問…？」、「想了解…？」的語氣，更自然不冒犯
