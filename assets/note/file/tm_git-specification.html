git規範、各分支定義及發佈版本流程

git規範

修改文件請使用標準的 commit 方式，請看這裡。

常用commit定義

feat - 新功能、項目
fix - bug修正
chore - 設定更新、更新版本、增加套件...
ci - ci設定變更
build - 專案建置、打包設定相關
refactor - 重構
BREAKING CHANGE - 會無法上下兼容

commit方式

feat([功能項名稱]): [改動了什麼]

e.g. feat(時間快捷): 遊戲匯總表增加時間快捷

各分支定義

  • develop
    • 主分支
    • 開發站使用
    • 開發都從此分支拉分支出去開發
  • release
    • 整合站使用
    • 從develop用cherry-pick的方式將要送測的項目pick過來
  • master
    • 正式站使用
    • 從master用cherry-pick的方式將要上線的項目pick過來
  • 自訂義分支
    • 分支命名規則
      • 新項目開發： feat/[名字]/[項目名] e.g. feat/black/add-account
      • bug修正： fix/[名字]/[項目名]
      • 重構： refactor/[名字]/[項目名]
      • 雜項(設定更新、更新版本、增加套件…)： chore/[名字]/[項目名]
